# **名大レゴ部 公式サイト再構築プロジェクト 仕様書**

## **1\. プロジェクト概要**

名大レゴ部（Nagoya University LEGO User Group）の公式サイトを、Antigravity（LLMアプリプラットフォーム）を活用して刷新する。本プロジェクトの核心は「管理の自動化」と「継続的な継承性」にある。

## **2\. サイト・コンセプト**

* **デザイン:** Google/Appleのような「シンプルかつ華のあるUI」。作品が主役となるクリーンな構成。  
* **運用思想:** 「部員が特別なIT知識なしで更新できる」こと。Google WorkspaceをバックエンドのDB（データベース）として活用する。

## **3\. 機能要件**

| 機能 | 詳細仕様 | 連携ツール   |
| :---- | :---- | :---- |
| 自動更新コラム | Googleフォームへの投稿内容をAIが整形し、活動報告や作品紹介として自動掲載。 | Google Forms / Antigravity (LLM) |
| 作品ギャラリー | Studio 2.0のレンダリング画像や実物写真をタグ（カテゴリ）別で表示。 | Google Drive / Make |
| SNSフィード | 公式X（旧Twitter）およびInstagramの最新投稿を表示。 | 各SNS公式API / Widget |
| 案件受付フォーム | 外部からの制作・出展依頼を収集。回答はDiscordの特定チャンネルへ即時通知。 | Google Forms / Discord API |
| 引き継ぎマニュアル | サイトの構造やAPI連携キーの管理方法を明文化。 | Google Docs (V1) |

## **4\. システム・アーキテクチャ**

将来的なメンテナンス性を考慮し、以下の構成を採用する。

* **Frontend:** AntigravityによるWebアプリフロントエンド。  
* **Backend DB:** Google Sheets（全部員が編集可能なマスタデータ）。  
* **Automation:** GAS (Google Apps Script) および Make を利用したデータの自動同期。  
* **Output Hub:** Discordの特定チャンネルに更新情報を自動出力。

## **5\. 開発のステップ（余白）**

以下の項目については、実装を進めながら部内でのフィードバックを経て決定する。

* 作品写真の命名規則およびフォルダ構成の最適化。  
* AI校正時の「部らしさ」を定義するプロンプトの調整。  
* デザインにおける「木製パーツ」のようなアクセントの導入検討。

## **6\. 次のアクション**

1. Google SheetsによるDB構成の策定。  
2. GoogleフォームとAntigravityを連携させるAPIフローの構築。  
3. Discordへの自動通知テスト。
